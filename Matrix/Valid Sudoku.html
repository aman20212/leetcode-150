        Determine if a 9 x 9 Sudoku board is valid. Only the filled cells need to be validated according to the following rules:

        Each row must contain the digits 1-9 without repetition.
        Each column must contain the digits 1-9 without repetition.
        Each of the nine 3 x 3 sub-boxes of the grid must contain the digits 1-9 without repetition.
        Note:

        A Sudoku board (partially filled) could be valid but is not necessarily solvable.
        Only the filled cells need to be validated according to the mentioned rules.
        


        Help me with this question, along with time and space complexity.
    

    
        Rules Recap:
        ------------
        Each row must have unique digits 1‚Äì9.

        Each column must have unique digits 1‚Äì9.

        Each 3x3 sub-box must have unique digits 1‚Äì9.


        function isValidSudoku(board) {
            const rows = new Array(9).fill(0).map(() => new Set());
            const cols = new Array(9).fill(0).map(() => new Set());
            const boxes = new Array(9).fill(0).map(() => new Set());
          
            for (let i = 0; i < 9; i++) {
              for (let j = 0; j < 9; j++) {
                const val = board[i][j];
          
                if (val === '.') continue; // skip empty cells
          
                if (rows[i].has(val)) return false;   // check row
                if (cols[j].has(val)) return false;   // check column
          
                const boxIndex = Math.floor(i / 3) * 3 + Math.floor(j / 3);
                if (boxes[boxIndex].has(val)) return false; // check 3x3 box
          
                rows[i].add(val);
                cols[j].add(val);
                boxes[boxIndex].add(val);
              }
            }
          
            return true; // valid board
        }


        Time & Space Complexity
        --------------------------
        Time Complexity: O(1)

        Although we loop through 81 cells (9x9), it's considered constant (O(1)) since the board size is fixed.

        Space Complexity: O(1)
        --------------------------

        We use three arrays of 9 Sets each, all of constant size.


        What Are We Checking?
        ---------------------
        We need to make sure that every digit from '1' to '9':

        Appears only once per row

        Appears only once per column

        Appears only once per 3x3 box

        We can ignore empty cells (usually represented by '.').

Why Box Calculation ?

    
        Sudoku Rule Recap
        -----------------
        A valid Sudoku board must satisfy all three constraints:

        Each row contains digits 1‚Äì9 with no duplicates

        Each column contains digits 1‚Äì9 with no duplicates

        Each 3√ó3 sub-box contains digits 1‚Äì9 with no duplicates ‚úÖ ‚Üê this is the part you're asking about


        Why Check Boxes?
        ----------------
        Let‚Äôs say you only check rows and columns, and ignore boxes.

        Here‚Äôs a board that would pass the row and column checks, but is invalid due to the 3√ó3 box rule:


        5 3 . | . 7 . | . . .
        6 . . | 1 9 5 | . . .
        . 9 8 | . . . | . 6 .

        8 . . | . 6 . | . . 3
        4 . . | 8 . 3 | . . 1
        7 . . | . 2 . | . . 6

        . 6 . | . . . | 2 8 .
        . . . | 4 1 9 | . . 5
        . . . | . 8 . | . 7 9
        Let‚Äôs focus on the top-left box:


        5 3 .
        6 . .
        . 9 8

        If you insert another 5 in that 3√ó3 box, but not in the same row or column, row and column rules would still pass ‚Äî but the Sudoku is invalid, 
        because the box now has two 5s.

        üîç Example Violation (Hidden if No Box Check)
        Suppose we do:


        board[2][2] = '5';  // add 5 in top-left 3x3 box
        Now:

        Row 2: no duplicate ‚úÖ

        Column 2: no duplicate ‚úÖ

        But the box (top-left 3x3) has two 5s ‚ùå

        So the board becomes invalid ‚Äî but you'd miss that without checking the box.

        ‚úÖ Why 3 Checks Are Required
        To fully validate a Sudoku board, you must check:

        Constraint	Purpose
        Rows	Prevent duplicates in horizontal lines
        Columns	Prevent duplicates in vertical lines
        Boxes	Prevent duplicates in the 3√ó3 sub-grids

        Ignoring any one of these allows invalid boards to pass.
